### **--> What is Data \& Why It's Important**

* **Data Science:** fundamental paradigm within AI in which data is studied, transferred, \& uses to build predictive models
* **Structured \& Unstructured data, Semi-structured**
* data acts as the fuel for machine learning algorithms
* directly impacts model accuracy \& reliability
* without quality data even advanced models fails
* Data should be Relevance, Accuracy, Diversity \& Sufficient
* Role of data in ML Pipeline:

  * **Collection**: Gather data
  * **Pre-processing**: cleaning \& transforming
  * **Feature Engineering**: extracting useful features
  * **Model Training**: feeding data into algorithms
  * **Evaluation:** testing model performance





### **--> Introduction to NumPy, Pandas, Matplotlib \& Seaborn**

* **NumPy:** Mostly used for arrays

  * Library for numerical computations
  * Supports multi-dimensional arrays \& matrices
  * Enables efficient mathematical operations(linear algebra, statistics)
  * Key concepts: ndarray, Shape, ndim \& dtype
* **Pandas:** Used for matrices or multi-dimensional arrays

  * python library for data manipulation and analysis
  * provides Data Frame \& Series structures
  * Simplifies working with structured data(CSV, Excel, SQL)
  * **Series:** 1D labeled array, consists of index + value, automatically infers dtype
  * **DataFrame:** 2 dimensional labeled data structure in rows and columns, similar to a spreadsheet or SQL table, can hold different types in different columns
* **Matplotlib:** Widely used data visualization library

  * creates charts: line, bar, scatter, histograms
* **Seaborn:** Built on top of Matplotlib

  * Provides high-level, visually appealing plots
  * Ideal for statistical data visualization



### **--> EDA Key Questions \& Mindset**

* **EDA(Exploratory data Analysis)** is the process of understanding data before modeling
* Goals: identify patterns, trends \& anomalies
* check quality of data, like missing, duplicates, outliers
* understand features types, like numerical, categorical, text based
* be curious => ask why and how
* look for relationship between features 
* document assumptions \& findings
* **key takeaways:**

  * Always define goals before starting EDA
  * Identify targets \& features types early
  * use EDA questions to guide analysis
  * document data quality issues \& assumptions
